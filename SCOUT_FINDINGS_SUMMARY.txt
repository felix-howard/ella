QUESTION 3: The notification subscription model/schema
==============================================================================

DATABASE SCHEMA:
  - File: C:\Users\Admin\Desktop\ella\packages\db\prisma\schema.prisma

NotificationSubscription model:
  - id: cuid
  - subscriberId: Staff ID of person receiving notifications
  - targetStaffId: Staff ID of person being monitored
  - type: SubscriptionType (UPLOAD or CHAT)
  - Unique constraint: (subscriberId, targetStaffId, type)
  - Indexes on subscriberId and targetStaffId

SubscriptionType enum:
  - UPLOAD: Notify when monitored staff's clients upload documents
  - CHAT: Notify when monitored staff sends messages to clients

Staff model fields:
  - notifyOnUpload: boolean (global preference toggle)
  - notifyOnChat: boolean (global preference toggle)
  - notificationSubscriptions: relation to NotificationSubscription (as subscriber)
  - notificationSubscribers: relation to NotificationSubscription (as target)

LOGIC:
  - Admin receives chat notification IF:
    1. Admin.notifyOnChat: true (global preference enabled) AND
    2. Admin has NotificationSubscription(subscriberId=admin, targetStaffId=staff, type='CHAT')

SUBSCRIPTION MANAGEMENT API:
  - File: C:\Users\Admin\Desktop\ella\apps\api\src\routes\team\index.ts (lines 510-627)
  
  GET /team/members/:staffId/notification-subscriptions
    - Returns uploadSubscriptions[] and chatSubscriptions[] (array of target staff IDs)
    - Lists available members to subscribe to
  
  PUT /team/members/:staffId/notification-subscriptions
    - Replaces subscriptions for a type
    - Request: { targetStaffIds: string[], type: 'UPLOAD' | 'CHAT' }
    - Transaction: Delete old → Create new

SCHEMA VALIDATION:
  - File: C:\Users\Admin\Desktop\ella\apps\api\src\routes\team\schemas.ts (lines 38-42)
  - updateNotificationSubscriptionsSchema: { targetStaffIds: array of strings, type: enum }
